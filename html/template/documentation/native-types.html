RS/JSON supports the following basic types:

Name Code Minimum Maximum Precision
Float float -max max max

String string -max max

Boolean boolean 0 1

Date date
Time time
Date time date-time

Value list, like enum() or set() are not in RS/JSON native types because it is not a clean and maintainable solution for a set of values.

We prefer using a proper model with a primary key and a name field, and using this key to link your records to.
