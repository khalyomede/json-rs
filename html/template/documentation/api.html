  • version
If marked like <property>, it means this attribute can be found many times.

version

  • type string
A version number. To work better, mostly with migrations, you should support semantic versioning.

license

  • type string
  • default "none"
A license name.

charset

  • type string
  • default "utf8"
A character set to use for all the property inside the schema. See the charsets documentation for a list of supported character set.

schema

  • type object
  • default {}
Each key represent a model name

model

  • type object
A model represent a table in a database.

identifier

  • type array<string>
  • default []
A list of property name to use to identify the record.

uniques

  • type array<unique>
  • default []
A list of unique objects to use to add a constraint of the desired properties.

unique

  • type array<string>
A list of property name to use to specify the unique index.

properties

  • type object
A list of property.

property

  • type object
A set of attributes for a given model.

for

  • type string
A model name the current property relate to. Useful when the attribute type of the current property is defined to relation.

guards

  • type array<string>
  • default []
Guards perform validation before a data is being inserted or updated. Check out guards documentation for more information.

incremented

  • type boolean
  • default false
true will let the storage engine aware this is an auto incremented field.

minimum

  • type number | null
  • default null
Constraint the property to a certain length. If used with type: relation, it will apply a one to many, many to one, or many to many. To specify a many to *, emit this attribute or set it to null (wich means "infinity").

maximum

  • type number | null
  • default null
Constraint the property to a certain length. If used with type: relation, it will apply a one to many, many to one, or many to many. To specify a * to many, emit this attribute or set it to null (wich means "infinity").

precision

  • type number
  • default 0
Used with a number type (see types documentation), to give a specific precision (number of digits after the comma).

required

  • type boolean
  • default false
Specifiy that the current property should be present when inserting or updating a record of the current model.

type

  • type string
Specify which type the property relates to. Useful for controling the quality of the data inside your storage engine. Check types documentation for more information.

through

  • type string
Specify on which table (and not the model name!) the relation will pass through. Only useful when properties minimum and maximum are set to null or not mentioned, and applied when the attribute type is set to relation. Check types documentation for more information.
