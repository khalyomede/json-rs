Relational Schema over JSON

example.rs.json 
{
  "version": "0.1.0",
  "license": "MIT",
  "charset": "utf8-mb4",
  "schema": {
    "book": {
      "identifier": ["id"],
      "uniques": {
        "unique_book_name": ["name"]
      }
      "properties": {
        "id": {
          "type": "integer",
          "incremented": true
        },
        "name": {
          "type": "small-string",
          "required": true,
          "guards": ["filled"]
        }
      }
    }
  }    
}

Getting started

Single source of truth

RS/JSON is a standard proposal for relational schemas. The goal is to unify every softwares, like your relational database software, ORM and migration manager. It is a meta-language over all these tools, to help you focus only once on your database schema.

Raise your data quality

With the help of "guards", which are performing pre-insert/update validations on your software back-end, your data are ensured to be the cleanest possible. This is a good thing when planing on harvesting your data using data analysis tools.

Scalable

Designed to keep the same performance whether you have 10 or 1000 objects, thanks to JSON key-pair architecture.
Native types are not enough? Create your own types to abstract your work and focus on your business.
No guard fit your need? Create a customized one.
