Types define how you data is stored. RS/JSON provides natives types to help you kickstart your schema.

example.rs.json
{
  "version": "0.1.0",
  "license" : "MIT",
  "charset": "utf8",
  "schema": {
    "book": {
      "properties": {
        "name": {
          "type": "string"
        }
      }
    }
  }
}

Unless guards, types can actually alter other properties. This make those very powerful to abstract some business logic.

For example, instead of writing this:

example.rs.json
{
  "version": "0.1.0",
  "license": "MIT",
  "charset": "utf8",
  "schema": {
    "position": {
      "properties": {
        "longitude": {
          "type": "float",
          "precision": 8,
          "guards": ["min:-180", "max:180"]
        }
      }
    }
  }
}

You can write this:

example.rs.json
{
  "version": "0.1.0",
  "license": "MIT",
  "charset": "utf8",
  "schema": {
    "position": {
      "properties": {
        "longitude": {
          "type": "longitude"
        }
      }
    }
  }
}

Types like this are not native, but custom types. They are good if you want to reduce repetition.
